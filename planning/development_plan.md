# 생리 앱 개발 고려사항 정리

## 1. 개발 목표
- 바이브 코딩을 최대한 활용한다.
- 개발 과정에 skill, harness 같은 방식도 적극 적용한다.
- 빠르게 만들고, 빠르게 검증하면서 완성도를 높인다.
- 기획부터 개발, 배포까지 연결되는 흐름을 만든다.

## 2. MVP 범위
처음부터 모든 기능을 넣기보다, 핵심 가치를 보여주는 최소 기능부터 만든다.

우선순위 예시:
- 생리 기록
- 통증 입력과 해석
- 몸무게 입력과 해석
- 생리 지연 알림
- 홈 화면 개인화 정보 노출

## 3. 데이터 설계
- 사용자가 무엇을 언제 입력할지 기준을 정한다.
- 생리 시작일, 종료일, 통증, 체중, 관계 관련 기록을 어떤 구조로 저장할지 정한다.
- 단순 저장이 아니라 해석 가능한 형태로 데이터를 쌓는다.

## 4. 개인화 로직
- 기록값을 어떻게 해석해서 안내할지 규칙이 필요하다.
- 통증이 심할 때 병원 진료 권장 기준을 정한다.
- 체중 변화가 생리 중 자연스러운 범위인지 판단하는 기준을 정한다.
- 생리 지연 시 정상 범위와 주의 범위를 구분한다.

## 5. 알림 및 안내 방식
- 너무 자주 알리면 피로해지고, 너무 적으면 앱의 가치가 떨어진다.
- 사용자가 현재 상태를 이해할 수 있는 톤으로 안내해야 한다.
- 의학적 경고처럼 보이는 메시지는 신중하게 표현해야 한다.

## 6. 개인정보와 보안
- 생리 정보는 민감정보이므로 저장과 접근 방식을 신중하게 설계해야 한다.
- 계정, 기기 저장, 백업 여부를 초반에 결정해야 한다.
- 플레이스토어 배포를 생각하면 개인정보 처리방침도 함께 준비해야 한다.

## 7. 화면 구조
- 홈 화면에서 가장 중요한 정보를 먼저 보여준다.
- 기록 입력 화면은 최대한 빠르게 끝나야 한다.
- 정보성 콘텐츠는 카테고리별로 정리해서 보여준다.

## 8. 기술 및 구현 전략
- 기술 스택은 빠른 개발과 유지보수를 기준으로 선택한다.
- 자동화 가능한 부분은 skill, harness 같은 방식으로 붙여 개발 속도를 높인다.
- 초기에는 복잡한 기능보다 검증 가능한 기능 위주로 구현한다.

## 9. 검증과 배포
- 실제 사용자가 기록보다 도움을 느끼는지 테스트한다.
- 핵심 기능이 동작하는지 먼저 확인한 뒤 디자인과 콘텐츠를 다듬는다.
- 플레이스토어 배포를 위한 심사 기준, 권한, 문구를 미리 점검한다.
